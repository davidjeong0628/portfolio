HTML Learning Journey

HTML is a foundational component of web design. There are many technologies that are used to build websites: React, jQuery, Angular, Vue, etc. It may be tempting to learn these "hip" technologies first. However, it is wise to learn HTML first to truly understand how websites work. Furthermore, HTML is a powerful technology with many features. We will explore some of the cool features that HTML offers in this article.

In addition to HTML, CSS and JavaScript are also foundational technologies of web design.

To view the code for how these elements work, you can use your browser's developer tools to inspect elements or view the source code of this page.

The <picture> element can be used to load different images depending on the context. For example, it can be used to load different images based on the screen size or whether the browser supports a given image file format.

The <em> element, used for emphasis, can be used to change the meaning of a sentence. For example, "Really?" and "Really?" can be interpreted to have different meanings. Although it depends on the context, the normal "Really?" most likely will be interpreted as a sincere question, while the emphasized "Really?" may be interpreted as being sarcastic or greatly surprised.

The <b> element may look like it should be used to bold text. However, doing so is not good practice. Bolding text should be done through CSS. Rather, the <b> element should be used to draw attention to the content without adding any semantic importance to it. For example, MDN recommends using the <b> element to highlight things like keywords in a summary. For instance, HTML should be used for semantic purposes.

The <i> element may look like it should be used to italize text. However, as we saw with the <b> element, we cannot fall into this trap. Rather, <i> should be used to represent text that is somehow different different from the text. For instance, we are Homo sapiens.

The <br> element creates a line-break. This is useful when writing poems or addresses. For example, here's a made-up poem:

You looked me in the eye.
Goodbyes as usual.
I knew it'd be the last time.

The <pre> element keeps whitespaces. Usually, HTML only keeps a single whitespace. This can be useful when writing code: 

        for (let i = 0; i < 10; i++) {
          console.log("wow");
        }
        console.log("for-loop finished")

The <hr> element represents a thematic break. For example, a change of scene in a play:

John looked out into the Rockies with a burning desire to see the world as it is.

Ten years have past. John, sits in the driver seat, with no hope and desire but a burning cigarette between his fingers.

The <svg> element can be used for scalable vector graphics. For example, we can use this element to make a GitHub icon.

The <div> and <span> elements should be used with care. Both of them do not hold any semantic meaning. Thus, they should be used as a last resort when no semantically-meaningful element can be used. The two elements are often used to group content for styling purposes. The former is a block container, while the latter is inline.
